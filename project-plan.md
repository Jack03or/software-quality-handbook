# Project Plan

## Objective
The aim of this project is to create a Software Quality Handbook for a development team. The handbook will define best practices to improve consistency and reduce defects in projects.

## Topics
- Task Estimation in Scrum  
- Code Reviews  
- Defect Management  

## Approach
The project will be completed using a shared GitHub repository. Work will be done using feature branches and merged through pull requests. Team members will review each other's work before merging changes to the main branch. The repository will be updated gradually as the project progresses.

## Responsibilities
Each team member will take the lead on one topic and also contribute to another to ensure collaboration:

- Task Estimation: Jack (lead), Dylan  
- Code Reviews: Dylan (lead), Shane  
- Defect Management: Shane (lead), Jack  

Each topic will have at least two contributors.

## Tasks
- Research sources for each topic  
- Write handbook sections  
- Add diagrams where appropriate  
- Review each other's work before merging to main  
- Improve and refine content  
- Finalise formatting and structure  
- Complete retrospective  

## Timeline
- Day 1–2: Setup and planning  
- Day 3: Create outlines for each topic  
- Day 4: Research and gather sources  
- Day 5–6: Write initial drafts  
- Day 7: Review, feedback, and improvements  
- Day 8: Final checks, diagrams, and retrospective
