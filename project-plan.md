# Project Plan

## Objective
The aim of this project is to create a Software Quality Handbook for a development team. The handbook will define best practices to improve consistency and reduce defects in projects.

## Topics
- Task Estimation in Scrum  
- Code Reviews  
- Defect Management  

## Approach
The project will be completed using a shared GitHub repository. Work will be done using feature branches and merged through pull requests. Team members will review each other's work before merging.

## Responsibilities
Each team member will take the lead on one topic and also contribute to another:

- Task Estimation: Jack (lead), Dylan  
- Code Reviews: Dylan (lead), Shane  
- Defect Management: Shane (lead), Jack  

## Tasks
- Research sources for each topic  
- Write handbook sections  
- Add diagrams where needed  
- Review and improve each section  
- Finalise formatting and structure  
- Complete retrospective  

## Timeline
- Day 1–2: Setup and planning  
- Day 3–4: Research and outlines  
- Day 5–6: Writing and development  
- Day 7: Review and improvements  
- Day 8: Final checks and retrospective
